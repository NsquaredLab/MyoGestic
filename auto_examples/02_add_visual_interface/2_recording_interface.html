Part 2: Recording Interface#

The recording interface is the second component of a visual interface in MyoGestic.

It is responsible for managing the ground-truth data collection. Since the ground-truth data depends on the visual interface, the recording interface must define the data collection process.

Step 1: Create the setup UI#

You will need an UI that will allow the user to start the recording, visualize the progress, and save the data.

Note

To start copy the UI files in myogestic > gui > widgets > visual_interfaces > ui and adapt them with the functionality you need.

You need to modify them with QT-Designer and convert them using UIC to a python file.

Step 2: Understand what is needed in the recording interface#

Note

A recording interface is a class that inherits from

Please read the documentation of the class and make a mental note of what you have to provide (e.g. signals, methods, attributes) and what you have to implement (e.g. start_recording, stop_recording, accept_recording).

Step 3: Implement a recording interface (Example Virtual Hand Interface)#

This example focuses on implementing and adding the recording interface for the Virtual Hand Interface using the VirtualHandInterface_RecordingInterface class from recording_interface.py. We explain how it is constructed and registered into MyoGestic via CONFIG_REGISTRY in config.py.

Step 3.2: Starting & Stopping Recordings#

This step manages the logic for starting or stopping recording sessions, which includes preparing recording parameters and updating the UI.

Start Recording Preparation - This method prepares the recording session.#
141    def start_recording_preparation(self) -> bool:
142        """Prepares the recording process by checking if the device is streaming."""
143        if (
144            not self._main_window.device__widget._get_current_widget()._device._is_streaming
145        ):
146            self._main_window.logger.print(
147                "Biosignal device not streaming!", level=LoggerLevel.ERROR
148            )
149            return False
150
151        self.kinematics_recording_time = int(
152            self.record_duration_spin_box.value() * KINEMATICS_SAMPLING_FREQUENCY
153        )
154        self._kinematics__buffer = []
155        return True
Start Recording - This method starts the recording session.#
115    def start_recording(self, checked: bool) -> None:
116        """Starts the recording process."""
117        if checked:
118            if not self.start_recording_preparation():
119                self.record_toggle_push_button.setChecked(False)
120                return
121
122            if not self._recording_protocol.start_recording_preparation(
123                self.record_duration_spin_box.value()
124            ):
125                self.record_toggle_push_button.setChecked(False)
126                return
127
128            self._start_time = time.time()
129
130            self.record_toggle_push_button.setText("Recording...")
131            self.record_group_box.setEnabled(False)
132            self._current_task = self.record_task_combo_box.currentText()
133
134            if self.use_kinematics_check_box.isChecked():
135                self.incoming_message_signal.connect(self.update_ground_truth_buffer)
136
137            self._has_finished_kinematics = (
138                not self.use_kinematics_check_box.isChecked()
139            )

Step 3.3: Managing Recording Sessions#

This step manages the recording session, which includes updating the UI and recording data.

Update Ground Truth Buffer - This method updates the ground-truth buffer with the current data.#
157    def update_ground_truth_buffer(self, data: np.ndarray) -> None:
158        """Updates the buffer with the incoming kinematics data."""
159        if not self.use_kinematics_check_box.isChecked():
160            return
161
162        self._kinematics__buffer.append((time.time(), data))
163        current_samples = len(self._kinematics__buffer)
164        self._set_progress_bar(
165            self.record_ground_truth_progress_bar,
166            current_samples,
167            self.kinematics_recording_time,
168        )
169
170        if current_samples >= self.kinematics_recording_time:
171            self._main_window.logger.print(
172                f"Kinematics recording finished at: {round(time.time() - self._start_time)} seconds"
173            )
174            self._has_finished_kinematics = True
175            self.incoming_message_signal.disconnect(self.update_ground_truth_buffer)
176            self.check_recording_completion()
Check Recording Completion - This method checks if the recording session is complete.#
178    def check_recording_completion(self) -> None:
179        """Checks if the recording process is complete and finishes it if so."""
180        if (
181            self._recording_protocol.is_biosignal_recording_complete
182            and self._has_finished_kinematics
183        ):
184            self.finish_recording()
Finish Recording - This method finalizes the recording session.#
186    def finish_recording(self) -> None:
187        """Finishes the recording process and switches to the review recording interface."""
188        self.review_recording_stacked_widget.setCurrentIndex(1)
189        self.record_toggle_push_button.setText("Finished Recording")
190        self.review_recording_task_label.setText(self._current_task.capitalize())
Accept Recording - This method accepts the recorded data and saves it.#
192    def accept_recording(self) -> None:
193        """
194        Accepts the current recording and saves the data to a pickle file.
195
196        The saved data is a dictionary containing:
197
198        - emg: A 2D NumPy array of EMG signals with time samples as rows and channels as columns.
199        - kinematics: A 2D NumPy array of kinematics data (empty if not used).
200        - timings_emg: A 1D NumPy array of timestamps for EMG samples.
201        - timings_kinematics: A 1D NumPy array of timestamps for kinematics samples (empty if not used).
202        - label: The user-provided label for the recording.
203        - task: The task being recorded.
204        - device: The name of the device used for recording.
205        - bad_channels: A list of channels marked as "bad."
206        - _sampling_frequency: The EMG sampling frequency.
207        - kinematics_sampling_frequency: The kinematics sampling frequency.
208        - recording_time: The recording duration in seconds.
209        - use_kinematics: Boolean indicating whether kinematics data was recorded.
210        """
211        label = self.review_recording_label_line_edit.text() or "default"
212        (
213            biosignal_data,
214            biosignal_timings,
215        ) = self._recording_protocol.retrieve_recorded_data()
216
217        self.save_recording(
218            biosignal=biosignal_data,
219            biosignal_timings=biosignal_timings,
220            ground_truth=(
221                np.vstack([data for _, data in self._kinematics__buffer]).T
222                if self.use_kinematics_check_box.isChecked()
223                else np.array([])
224            ),
225            ground_truth_timings=(
226                np.array([time_stamp for time_stamp, _ in self._kinematics__buffer])
227                if self.use_kinematics_check_box.isChecked()
228                else np.array([])
229            ),
230            recording_label=label,
231            task=self._current_task,
232            ground_truth_sampling_frequency=KINEMATICS_SAMPLING_FREQUENCY,
233            use_as_classification=not self.use_kinematics_check_box.isChecked(),
234            record_duration=self.record_duration_spin_box.value(),
235        )
236
237        self.reset_ui()
238        self._main_window.logger.print(
239            f"Recording of task {self._current_task.lower()} with label {label} accepted!"
240        )
Reject Recording - This method rejects the recorded data and discards it.#
242    def reject_recording(self) -> None:
243        """Rejects the current recording and resets the recording interface."""
244        self.reset_ui()
